# 부패영향평가 산출 문서 작성 규칙

## 공통 원칙

1. **서식 원천**: 반드시 `get_form_template(서식명)`을 호출해 별지 서식 구조를 따른다
   (기초자료=별지1 / 결과통보서=별지2 / 세부평가서=별지3).
2. **결론 유형은 3종뿐**: 원안동의 / 개선권고 / 철회의견 (지침 제2조②). 임의 유형 금지.
3. **증거 인용 형식**:
   - 자체규정: 「규정명」 제○조제○항 (개정일)
   - 타 기관: 기관명 「규정명」 제○조 — fetch_peer_rule로 확인한 본문만 인용
   - 법령: 「법령명」 제○조 (시행일자, MST) — get_law_text로 확인한 조문만 인용
   - 지침 기준: 별표1 기준명 / 별표3·4 체크항목 원문
4. **문체**: 정부 공문서 개조식 — □(쟁점) → ㅇ(검토 내용) → -(부연). 음슴체("~함", "~필요").
5. **수치·날짜**: 확인된 값만. 미확인은 "(확인 필요)"로 명시 — 빈칸을 추정으로 채우지 않는다.

## 세부평가서 (별지3) — 조문 단위 작성

쟁점 조문마다 한 블록:

```
## 평가대상 조문: 제○조(제목) — [신설/개정/현행유지]

□ 평가기준: (별표1의 해당 기준명 — 복수 가능)

□ 현황
 ㅇ (현행) 현행 조문 요지 — 신·구 대비
 ㅇ (개정안) 개정안 요지

□ 문제점
 ㅇ (쟁점) 부패유발요인 분석 — 해당 체크항목 명시
   - 비교 증거: 타 기관 수준·상위법령 범위·자체규정 충돌 등 도구로 확보한 사실
 ㅇ (소극행정) 명확성·구체성 부재로 인한 소극행정 유발 가능성 검토 (제6조① 후단)

□ 검토결과
 ㅇ (의견안) 수정·보완·삭제 등 구체적 개선안 — 권고 문안 형태로
```

## 결과통보서 (별지2)

- 검토결과 란: 원안동의/개선권고/철회의견 중 택1 + 관련조문 열거
- 개선권고·철회의견이면 세부평가서 첨부를 명시
- 말미 고지: 미반영 시 권고일로부터 5일 이내 사유 통보 의무(제8조②),
  의견 상이 시 최종결재권자에게 양측 의견 첨부(제8조③)

## 마무리 체크

- [ ] 모든 인용에 출처가 있는가 (도구 호출로 확인된 것만인가)
- [ ] 결론이 3유형 중 하나로 명확한가
- [ ] "본 평가서는 AI 보조 검토 초안이며, 최종 판단·확정은 평가담당자 결재로 한다" 문구를 말미에 부기
