# 부패영향평가 AI (corruption-impact-ai)

사규 제·개정안의 부패유발요인을 **입안 단계에서** 검토할 수 있도록 돕는
**대화형 사전검토 보조도구** — Claude Code 플러그인입니다.

> 자동 평가기가 아닙니다. **판단은 담당자가, 분석은 Claude가, 자료는 MCP가** 맡습니다.
> 결과만 내놓는 자동화로는 평가에 책임을 싣기 어렵기 때문에, 담당자가 대화로 검토를
> 진행하고 모든 인용에 출처를 남기며, 최종 확정은 담당자 결재로 합니다.

쉽게 말하면 — AI(Claude)에 **규정·법령·타 기관 자료를 실시간으로 찾아다 주는
'전용 도구'(MCP)** 를 붙이고, 사용법(`/사규검토` 명령·작성 스킬)까지 한 벌로 묶은 것입니다.

| 주체 | 역할 |
|---|---|
| 담당자(사람) | 검토 방향 결정, 조문별 쟁점 질의, 최종 판단(원안동의/개선권고/철회의견) |
| Claude | 체크리스트 항목별 추론, 조문 대조 분석, 세부평가서 초안 |
| MCP(이 플러그인) | 평가기준·체크리스트 공급, 자체규정 검색, 타 기관 규정·경영지침·법령 조문 조달 |

## 왜 만들었나 — 지침이 요구하는 "비교"는 수작업으로 불가능하다

「부패영향평가 지침」 체크리스트는 *"유사 법령의 준수부담과 **비교**하라"*,
*"적용 범위가 **상위법령** 범주 내인가"*, *"타 규정과의 **저촉·중복**은 없는가"* 를 요구합니다.
전부 다른 규정·법령과의 대조가 전제인데, 종전에는:

| 지침 요구 | 종전 수작업 | 이 플러그인 |
|---|---|---|
| 타 기관과 형평성 비교 | 기관별 HWP를 하나하나 수동 다운로드 | **알리오 결합** — 전국 공시기관(약 355곳) 동종 규정 검색 → 본문 즉시 조달 |
| 상위법령 범주 검토 | 법제처 사이트 별도 검색 | **법제처 결합** — 현행 법령·행정규칙 조문 단위 조회 |
| 상위 경영지침 검토 | 알리오 게시판 수동 검색 | **경영지침 결합** — 공운법 제50조 지침 78건 조문 대조 (법제처 미등재 자료) |
| 자체규정 충돌·이중부담 | 규정 전문 137건을 기억에 의존해 대조 | **전문 코퍼스** — 137건 200만 자 키워드 전수 검색 |

평가 기준 자체도 내장했습니다 — 평가대상 게이트(별표2), 평가기준 11개(별표1),
체크리스트 164항목(별표3·4), 별지 서식 4종.

## 실제 검토 장면 (2026.5. 「임직원 행동강령」 개정안 — 실호출 결과)

- 개정안이 인용한 **별표3의3이 본문에 존재하지 않음**을 도구가 확인 — 존재하지 않는 구간
  인용을 입안 단계에서 적발
- 전국 행동강령 **403건**과 비교 — 특정 의무조항 보유율을 수 분 만에 전수 집계
  (종전엔 반나절 수작업)
- 자체 규정 간 **동일 문구 중복**(행동강령 ↔ 인사규정시행세칙) 탐지 → 정비 쟁점화

## 설치 (Claude Code)

```bash
git clone https://github.com/giovinazo/corruption-impact-ai.git
claude plugin marketplace add ./corruption-impact-ai
claude plugin install corruption-impact-ai@corruption-impact-ai --scope user
```

요구사항: Python 3.10+ / `pip install -r mcp/requirements.txt`
(mcp·requests·urllib3·olefile·PyMuPDF·python-docx)

Windows 비전문가 배포용으로는 **무설치 키트 빌더**(`tools/build_win_kit.py`)를 제공합니다 —
임베디드 Python과 의존성 전체를 동봉해, 대상 PC에 파이썬 설치 없이 동작합니다
(자가진단 `doctor.py`·`test_runtime.bat` 포함).

| `CIA_DEFAULT_PEERS` | — | 타 기관 기본 비교군 오버라이드 (쉼표구분) |

└── mcp/
    ├── server.py                # FastMCP 도구 16종
    ├── alio_client.py           # 알리오 클라이언트
    ├── law_client.py            # 법제처 클라이언트 (중계 프록시 경유)
    ├── hwp_extract.py           # HWP 스펙준수 파서 + HWPX/PDF/DOCX/HWPML
    ├── doctor.py                # 배포 프리플라이트 진단 (대상 PC 자가점검)
    ├── self_check.py            # 도구 검증 (PASS 29)
    └── data/
        ├── assessment_kb.json      # 지침 구조화 — 기준 11·체크리스트 164항목·서식 4종

        ├── guidelines_corpus.json  # 재정경제부 경영지침 78건 (현행 30/연혁 48)

## 데이터 출처·주의

- 내부규정 코퍼스·타 기관 규정·경영지침: 모두 **알리오(공공기관 경영정보 공개시스템)
  공개 공시 데이터**입니다. 비공개 자료는 포함하지 않습니다.
- 경영지침: 재정경제부가 공운법 제50조에 따라 정해 통보하는 지침(법제처 미등재) —
  알리오 '공공기관 법령/지침' 게시판 78건, 현행/연혁 자동 분류 (`tools/build_guidelines.py`)
- 「부패영향평가 지침」: 알리오 공시 최신본(2024.3.18 개정) 구조화
- **검토 시점 현행성(자가갱신)**: 검토 세션 시작 시 내장 코퍼스를 알리오 현행 공시와
  대조하고, 신규 공시가 있으면 사용자 확인 후 바뀐 항목만 증분 갱신합니다.
  모든 인용에 기준일·출처를 표기하며, 외부 호출 실패 시에도 검토를 중단하지 않고
  보유 데이터 기준임을 명시한 채 계속합니다(우아한 강등).
- 산출 문서는 **AI 보조 검토 초안**입니다 — 최종 판단·확정은 평가담당자 결재로 합니다.
